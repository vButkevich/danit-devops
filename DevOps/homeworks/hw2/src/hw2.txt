Роль і значення компонентів архітектури ОС Linux
a) Kernel (Ядро)
Ядро — це центральний компонент операційної системи Linux, який забезпечує взаємодію між апаратним забезпеченням та програмним забезпеченням.

Основні функції ядра:
Управління процесами:
     Планування виконання процесів (CPU scheduling)
     Розподіл процесорного часу між програмами
     Створення та завершення процесів
Управління пам'яттю:
     Розподіл оперативної пам'яті між процесами
     Віртуальна пам'ять (swap)
     Захист пам'яті процесів
Управління пристроями:
     Драйвери для апаратного забезпечення
     Взаємодія з периферійними пристроями
     Обробка переривань
Файлова система:
     Організація даних на дисках
     Підтримка різних файлових систем (ext4, NTFS, FAT32)
     Керування правами доступу
Мережева підтримка:
     Реалізація мережевих протоколів (TCP/IP)
     Маршрутизація пакетів
     Мережева безпека
Приклад:bash
# Перегляд версії ядра
uname -r
# Результат: 5.15.0-91-generic
b) Libraries (Бібліотеки)
Бібліотеки — це набори готових функцій та процедур, які програми можуть використовувати для виконання типових завдань, не "винаходячи велосипед".

Типи бібліотек:
Стандартна бібліотека C (glibc)
Найважливіша бібліотека в Linux
Містить базові функції: введення/виведення, робота з рядками, математичні операції
Забезпечує системні виклики до ядра
Динамічні бібліотеки (.so)
Завантажуються в пам'ять під час виконання програми
Економлять пам'ять (одна бібліотека для багатьох програм)
Можна оновлювати без перекомпіляції програм
Статичні бібліотеки (.a)
Вбудовуються прямо в програму при компіляції
Більший розмір програми, але незалежність від системи

Приклад:bash
# Перегляд залежних бібліотек програми
ldd /bin/ls
# Результат покаже всі бібліотеки, які використовує ls
c) System Utilities (Системні утиліти)
Системні утиліти — це програми, які виконують специфічні завдання з управління системою та надають інтерфейс для взаємодії користувача з ОС.

Категорії системних утиліт:
Управління файлами
ls, cp, mv, rm, mkdir
Навігація та маніпуляція файловою системою

Управління процесами
ps, top, kill, nice
Моніторинг та керування виконанням програм

Адміністрування системи
useradd, passwd, chmod, chown
Налаштування користувачів та прав доступу

Мережеві утиліти
ping, ifconfig, netstat, ssh
Діагностика та налаштування мережі

Оболонка (Shell)
Bash, Zsh, Fish — інтерпретатор команд
Забезпечує інтерактивну взаємодію з системою

Приклад:bash
# Використання системних утиліт
ps aux          # Перегляд процесів
df -h           # Інформація про диски
grep "error" log.txt  # Пошук тексту

Взаємодія компонентів
text
Користувач
    ↓
[System Utilities] ←→ [Libraries]
         ↓                   ↓
    [System Calls] ←→ [Kernel]
                            ↓
                      [Hardware]

Схема взаємодії:
Користувач вводить команду в терміналі
System Utility (наприклад, ls) обробляє команду
Утиліта використовує Libraries (функції для роботи з файлами)
Бібліотеки роблять System Calls до ядра
Kernel виконує операції з апаратним забезпеченням
Результат повертається назад до користувача

Значення кожного компонента
Компонент	        Значення
Kernel	            "Мозок" системи — безпосереднє управління апаратурою, основа стабільності та безпеки
Libraries	        "Будівельні блоки" — прискорюють розробку, забезпечують сумісність, економлять ресурси
System Utilities	"Інструменти" — роблять систему корисною для людини, надають зручний інтерфейс
Ключовий принцип: модульність — кожен компонент відповідає за свою частину, що робить систему гнучкою, надійною та легкою в обслуговуванні.
