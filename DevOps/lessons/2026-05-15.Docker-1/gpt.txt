Покажу практичний варіант без Docker Desktop: окремо для Linux VM і для Windows Server VM, бо інструкції сильно відрізняються. Основна ідея: ставимо Docker Engine, а не Docker Desktop.

Thought for 6s
Найкращий варіант для VM без Docker Desktop — ставити Docker Engine напряму в ОС. Docker Desktop не потрібен.

1. Рекомендований варіант: Linux VM Ubuntu/Debian
Для серверного використання це найпростіший і найстабільніший варіант.

Docker офіційно підтримує Docker Engine на Ubuntu/Debian через apt repository. Для Ubuntu підтримуються, зокрема, Ubuntu 24.04 LTS і 22.04 LTS; для Debian — Debian 13, 12, 11. 

Встановлення на Ubuntu
У VM відкрий terminal:

sudo apt-get update
Видали старі/конфліктні пакети, якщо вони є:

sudo apt-get remove -y docker.io docker-compose docker-compose-v2 docker-doc podman-docker containerd runc
Додай офіційний Docker repository:

sudo apt-get install -y ca-certificates curl

sudo install -m 0755 -d /etc/apt/keyrings

sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  -o /etc/apt/keyrings/docker.asc

sudo chmod a+r /etc/apt/keyrings/docker.asc

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}") stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
Встанови Docker Engine + Docker Compose plugin:

sudo apt-get update

sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
Перевір:

docker --version
docker compose version
sudo docker run hello-world
Docker Compose зараз краще використовувати як plugin через команду:

docker compose up -d
а не старий варіант:

docker-compose up -d
Docker офіційно описує Compose plugin як рекомендований варіант для Linux через пакет docker-compose-plugin. 

2. Щоб запускати Docker без sudo
sudo usermod -aG docker $USER
Після цього треба перелогінитись у VM або виконати:

newgrp docker
Перевір:

docker run hello-world
3. Приклад використання Docker
Створи папку:

mkdir test-docker
cd test-docker
Створи docker-compose.yml:

services:
  nginx:
    image: nginx:latest
    container_name: test-nginx
    ports:
      - "8080:80"
    restart: unless-stopped
Запусти:

docker compose up -d
Перевір:

docker ps
У браузері з host-машини відкрий:

http://<IP_VM>:8080
IP VM можна подивитися так:

ip a
або:

hostname -I
4. Налаштування мережі VirtualBox
Щоб з host-комп’ютера бачити контейнер на VM, краще поставити для VM:

Варіант A — Bridged Adapter
VirtualBox → VM → Settings → Network → Adapter 1:

Attached to: Bridged Adapter
Name: твоя мережева карта / Wi-Fi / Ethernet
Тоді VM отримає IP у твоїй локальній мережі, наприклад:

192.168.1.50
І з host можна буде відкрити:

http://192.168.1.50:8080
Варіант B — NAT + Port Forwarding
VirtualBox → VM → Settings → Network → Adapter 1:

Attached to: NAT
Advanced → Port Forwarding
Додай правило:

Name: nginx
Protocol: TCP
Host IP: 127.0.0.1
Host Port: 8080
Guest IP: 
Guest Port: 8080
Після цього з host відкривай:

http://localhost:8080
5. Якщо VM — Windows Server 2022/2025
Тут важливий момент: Docker Desktop не потрібен і зазвичай не використовується на Windows Server. Для Windows Server ставлять Docker Engine / Mirantis Container Runtime / Windows Containers.

Але є обмеження:

Windows Server Docker підходить для Windows containers
Тобто можна запускати Windows-based images, наприклад:

mcr.microsoft.com/windows/servercore
mcr.microsoft.com/windows/nanoserver
Але звичайні Linux images типу:

nginx
postgres
redis
ubuntu
sqlserver-linux
напряму на Windows Server Docker Engine не запустяться без Linux VM/WSL2/іншого Linux host.

Microsoft має окрему документацію для конфігурації Docker Engine на Windows Server. 

Windows Server: базове встановлення Containers feature
В PowerShell від Administrator:

Install-WindowsFeature containers
Restart-Computer
Після reboot можна ставити Docker provider:

Install-Module -Name DockerMsftProvider -Repository PSGallery -Force
Потім Docker:

Install-Package -Name docker -ProviderName DockerMsftProvider -Force
Restart-Computer
Перевір:

docker version
docker info
Тест Windows container:

docker run mcr.microsoft.com/windows/nanoserver:ltsc2022 cmd /c echo Hello from Windows container
Для Windows Server 2025 можуть бути нюанси з версіями образів. Образ контейнера має відповідати або бути сумісним з версією Windows host.

6. Що вибрати у твоєму кейсі
Якщо ціль — запускати типові dev/server контейнери:

SQL Server Linux
PostgreSQL
MongoDB
Redis
Nginx
.NET Web API
React build
RabbitMQ
Metabase
тоді краще:

VirtualBox VM з Ubuntu Server 24.04 LTS
+
Docker Engine
+
docker compose
Якщо VM вже Windows Server 2025 і ти хочеш саме Linux-контейнери, то найпростіше не мучити Windows Server, а створити окрему Ubuntu VM у VirtualBox і поставити Docker там.

Мінімальний чекліст
docker --version
docker compose version
docker ps
docker run hello-world
Приклад запуску:

docker run -d --name web -p 8080:80 nginx
Перевірити:

curl http://localhost:8080
З host-машини:

http://<IP_VM>:8080
Мій практичний висновок: для твоєї VirtualBox-інфраструктури найкраща схема — Ubuntu Server VM + Docker Engine, а Windows Server VM залишити для IIS/SQL Server/.NET hosting.
