# 🧪 Lab: Deploy Python App on EC2 with systemd + Nginx Reverse Proxy

## 🎯 Goal

You are provided with an **EC2 instance (Ubuntu)** and SSH access.

Your task is to:

1. Deploy a Python web application
2. Run it as a **systemd service**
3. Configure **Nginx as a reverse proxy**
4. Make the application accessible via port **80**

---
# 🧱 Architecture
Browser → Nginx (port 80) → Python App (port 8000 via systemd)


---

## 🧱 Your Tasks

### 1. Setup Environment

On EC2 instance:

- Install required packages:
- Python3
- pip
- venv
- nginx

---

### 2. Deploy Application

- Copy application to: /opt/myapp
- Create Python virtual environment
- Install dependencies from: requirements.txt

### 3. systemd Service

Create a systemd service: /etc/systemd/system/myapp.service

Requirements:

- Runs application using virtualenv Python
- Runs on boot automatically
- Restarts automatically on failure
### 4. Nginx Reverse Proxy

Configure Nginx to:

- Listen on port `80`
- Forward traffic to application
